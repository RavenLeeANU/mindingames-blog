对于技术人员，经常写博客是一个好习惯，一方面可以在忘记时拿出来查看以备不时之需，另一方面用文字输出也是检验自己是否真的掌握了知识的一个重要渠道，还能够在一些岗位的面试作为一个加分项。我过去也有写博客的习惯，使用过的博客平台包括**CSDN**，**简书**；自己也搭建过博客，但随着时间推移这个习惯并没有保留下来，一来是写博客通常比较费事，由于涉及到可能有人可能会看，因此想尽量把来龙去脉都写清楚，写作的成本很高，不如在**word**上直接记录，二来各个平台对markdown语法、图片格式等的处理不一样，有些公式的markdown来回折腾感觉有些得不偿失，后来只留下了本地保存的一些副本。但是时间一长依然存在弊端，虽然目前的大部分软件如**typora**，**onenote**或者**飞书**都支持了云同步，但是查找和管理时依然比较麻烦，但这一部分原因也是我没有很好的将文件归档造成的，缺乏了监督和仪式感确实不容易写出好的东西，以至于时间一长自己的笔记记在哪里，写的是什么都忘记了，又需要重新学习，因此这次还是下了决心自己搭建一个博客，同时还可以将后续要做的项目都整合在同一个地方。

---
### 搭建个人博客

搭建个人博客的因为前端技术的发展而门槛越来越低，即使不会编程也可以搭建出专业的博客网站。低代码甚至无代码的产品搭建框架也是比比皆是，像**WordPress**、**Substack**都可以让你在短时间内快速无痛部署。而对于想要有一定个性化定制需求的人来说，使用便捷的前端web框架例如**Next.js**,**bootstrape**等也能够快速搭建，此外**github**上还有大量的开源博客模板供人选择。本博客的搭建也是得益于timlrx这位作者的贡献，在此表示感谢。

https://github.com/timlrx/tailwind-nextjs-starter-blog 

下面则以搭建此博客为例记录下搭建一个本站类似的博客网站必须了解的基础概念和所需步骤。

### 基本内容

**一个简单网站的基础框架：前端页面+后端服务+数据库**

所选技术栈 
- Next.js for Web fullstack
- TailwindCSS 页面样式框架
- Vervel 博客站点Serverless部署
- *bonus Dynadot域名注册

### 事前准备

#### 购买域名
可以通过域名供应商购买域名，需要注意的是在域名部署在国内服务器上时需要通过备案，而在国外服务器则不需要。在国内请求国外服务器的域名也是可以请求到的，但是不保证稳定性。
我选择是在[Dynadot](https://www.dynadot.com/)上注册的域名，这个供应商支持中国国内的VISA和MasterCard的支付。

#### 博客写作
平时我比较喜欢使用飞书先记录文稿的内容，再转换为Markdown的形式。虽然这么做会有点麻烦，但也算是一个帮我复习和整理一下看过的内容的契机。
飞书本身不支持导出Markdown文稿，直接复制会导致格式错乱，图片的路径也需要修改，找了一圈发现[Wsine/feishu2md](https://github.com/Wsine/feishu2md)这个工具比较好用。在windows下可以通过命令行执行导出Markdown文件。

具体的使用方式可以看[这里](https://zhuanlan.zhihu.com/p/650158515)

### 开发准备

#### Fork博客到我的仓库并clone到本地
```git clone https://github.com/RavenLeeANU/mindingames-blog.git``` 

#### 配置IDE与环境
需要安装的东西有包括：
- IDE : VS Code
    - 安装TailwindCSS的提示插件 : Tailwind CSS IntelliSense
        - 插件本身安装完成之后需要把Emmet Completions这一项勾选才能在coding时显示提示
        - 注：如果插件配置找不到就重启VS Code或者重启电脑

- 安装Node.js
    - 用>=19版本的Node，太低了Next.js不支持 [Node.Js Download](https://nodejs.org/en/download/current)

- 安装工程依赖
```yarn```

- 添加 Windows下的环境变量
```set PWD="$(pwd)"```
注：如果IDE的Terminal不能正常设置，就直接在CMD执行

- 启动项目
```yarn dev```

### 语法检查

- 换行等语法问题会影响构建，需要在提交时使用语法检查，blog框架已经写好了语法检查命令，直接执行即可。
```npm run lint```

### 构建部署

使用Serverless服务可以帮我们在不需要购买服务器的情况下快速部署网站，而且是免费的。
博客本身支持了多种部署方式，这里我们选择Vercel来进行部署。

- [Vercel](https://vercel.com/)关联Github
关联Github账户后，可以支持从关联的账户仓库导入项目，点击【Add New Project】后直接导入项目进行构建。

- 绑定域名
切到工程中选择 Add Domain


如果想要添加次级域名，可以按照如下格式进行命名
```subdomain.domain.com```
```https://generator.project.mindingames.com/```
